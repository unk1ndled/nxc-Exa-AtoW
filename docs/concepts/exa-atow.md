# Where this fits in Exa-AToW

[Exa-AToW](https://numpex.org/exaatow-architectures-and-tools-for-large-scale-workflows/)
stands for *Architectures and Tools for Large-Scale Workflows*. It is one of
the targeted projects in the French
[NumPEx](https://numpex.org/) programme. Its scope is much larger than one
cluster: scientific workflows increasingly span instruments, edge systems,
data platforms, AI services, and HPC facilities. Coordinating computation and
data across that continuum raises questions about interoperability, resource
federation, security, energy, and sustainability.

This repository explores one concrete slice of that problem:

> Can a scientific application be exposed through service and data APIs,
> translated into a conventional scheduler job, and exercised on a
> reproducible local cluster before it reaches a real facility?
